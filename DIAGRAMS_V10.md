# Graph Studio v10: интерактивные диаграммы

## Общий принцип

Все режимы выбираются через `diagram.type` и общий метод:

```js
graph.setDiagramType('fishbone');
```

Смена режима не ограничивается заменой раскладки. Модуль `diagram-interpreter.js` преобразует актуальное содержание проекта под семантику целевой диаграммы.

## Таблица режимов

| ID | Представление | Интерактивность | Основные данные |
|---|---|---|---|
| `network` | Сетевая диаграмма | вращение, zoom, drag, jelly | `nodes`, `links` |
| `force` | Силовой граф | физика, drag, стабилизация | `nodes`, `links` |
| `tree` | Иерархия | drag, hover, ручная позиция | корень и уровни |
| `mindmap` | Ментальная карта | drag ветвей, hover | ядро и ветви |
| `fishbone` | Исикава | pulse, flow, drag причин | проблема, категории, причины |
| `flowchart` | Блок-схема | drag блоков, hover переходов | формы узлов и связи |
| `decision` | Граф решений | drag, hover, формы сущностей | decision/chance/outcome |
| `sankey` | Санкей | hover узлов и лент | `column`, `link.value` |
| `bubble` | Пузырьковая | drag пузырей | `x`, `y`, `value` |
| `radar` | Радарная | drag вершин серии | `chart.metrics`, `chart.series` |

## Read-only

В режиме пользователя остаются только безопасные взаимодействия просмотра. Контекстные DOM-окна и операции изменения данных не создаются.

## Демонстрационные данные

Кнопка «Загрузить пример этого типа» использует `src/diagrams/templates.js`. Шаблоны намеренно насыщены, чтобы проверять не только первый кадр, но и длинные ветви, перекрёстные связи, сильные потоки, несколько серий и разные размеры элементов.
