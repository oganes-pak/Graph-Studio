# Graph Studio v8: руководство по коду

## Органическая пульсация

`heart-pulse.js` вычисляет только фазу удара. `organic-shapes.js` превращает
интенсивность в мягкую деформацию контура. При стиле `organic` отключаются
Canvas-shadow и светящиеся кольца. Узел немного сжимается и меняет форму,
а по связи проходит каплевидный маркер.

## Подписи ядер

`Graph3DEngine.drawNodeLabel()` поддерживает режимы:

- `none`;
- `core`;
- `all`.

Настройки находятся в `config.node.labels`.

## Всплывающие окна

Canvas больше не рисует информационную карточку в основном приложении.
`NodeInfoController` создаёт DOM-карточку, а `OverlayManager` выбирает позицию
с минимальным пересечением с легендой, toolbar и status bar.

`LinkEditorController` зарегистрирован в той же группе оверлеев. Поэтому
карточка узла и редактор связи не могут быть открыты одновременно.

## Режим viewer

`editor.mode = "viewer"` и `editor.locked = true` являются разными, но
согласованными признаками. UI скрывается на уровне приложения, а движок
независимо блокирует методы изменения. Это не серверная модель безопасности.

## Машинный контракт

- Декларативный путь: `src/model/ai-plan.js`.
- Атомарный путь: `src/model/tool-registry.js`.
- Transport MCP: `mcp/server.mjs`.

Transport не определяет имена команд. Он только публикует общий реестр.

## Черновик и фиксация настроек v8

`LivePreviewController.preview()` вызывает `engine.updateConfig(...,
{ transient: true })`. Canvas сразу показывает результат, но контроллер
приложения не сохраняет transient-событие в LocalStorage.

`LivePreviewController.commit()` повторно применяет итоговый патч как обычное
изменение. Только после этого проект считается зафиксированным.

## Толщина связи v8

`buildLinkRibbonGeometry()` получает центральную толщину связи и экранные
радиусы двух узлов. В середине сохраняется сила связи, а ширина концов
ограничивается радиусом соответствующего узла. Поэтому маленький узел не
перекрывается чрезмерно толстой линией.
