# Graph Studio v11: выбор диаграммы и показательные демо

## Главное правило

Graph Studio старается интерпретировать любой набор данных под любой из десяти режимов. При этом техническая возможность построить изображение не гарантирует идеальную смысловую совместимость.

Интерфейс показывает оценку:

- 80–100%: естественное представление;
- 55–79%: условное представление;
- ниже 55%: сильное автоматическое преобразование.

## Демо-сценарии

| Тип | Демо | Что демонстрирует |
|---|---|---|
| Network | Жизненная экосистема | много уровней, поперечные связи, области жизни |
| Force | Запуск интернет-магазина | плотная сеть людей, сервисов и задач |
| Tree | Учебная программа ИБ | строгая иерархия из четырёх уровней |
| Mind Map | Личный план на год | свободные ветви учёбы, работы, здоровья и финансов |
| Fishbone | Причины ухудшения здоровья | проблема, категории, причины и подпричины |
| Flowchart | Оформление заказа | процессы, решения, повтор оплаты и возврат |
| Decision | Путь после университета | варианты, вероятностные события и исходы |
| Sankey | Путь покупателей | количественные потоки по восьми колонкам |
| Bubble | Ассортимент магазина | маржа, спрос и выручка товаров |
| Radar | Баланс жизни | четыре состояния по восьми общим метрикам |

## Автоматически выводимые данные

- Для Bubble без X/Y используются уровень, связность и порядок узлов.
- Для Radar без метрик используются входящие, исходящие, степень, размер и глубина.
- Для Sankey без `value` применяется значение `1`.
- Для Fishbone первый уровень становится категориями, остальные элементы причинами.
- Для Tree выбирается корень и формируется остовная иерархия.
- Для Flowchart форма блока определяется по входящим и исходящим переходам.
